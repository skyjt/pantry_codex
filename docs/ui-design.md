## 1. 设计基调

**参考微信桌面版的设计语言**：三栏结构、大量留白、极轻的分割线、克制的色彩。"简约优雅、克制但不老气"落到具体手段上：

- **一个主题色**点缀全局（按钮、选中态、自己的气泡底色），其余全部中性灰阶；主题色定为**茶青 `#3D8B6B`**（决议 #1）。
- **系统字体栈**：`PingFang SC / Microsoft YaHei / Noto Sans CJK`，正文 14px，绝不用衬线、不用艺术字。
- **图标统一为项目内自绘线性 SVG**（1.6px 描边、圆角端点、继承当前文字色），不混用拟物/彩色图标，不用 emoji 充当系统图标；设置图标必须画成清晰齿轮，不使用放射状“太阳”隐喻。Win7 / UOS 缺字平台不可靠，头像和内置 emoji 子集的可见形态使用 Twemoji 本地 SVG 子集兼容渲染，输入框编辑态也必须覆盖，协议与复制仍保留原始 emoji 字符。
- **品牌 logo 使用统一的“茶杯气泡”标识**：杯身兼具对话气泡轮廓，右侧保留杯把，上方两笔蒸汽；菜单栏用单色模板图并在 32px 画布内留透明边距，界面小 logo 用茶青 + 暖茶色，空状态 / 大图标用同一轮廓置于浅底图标面内（决议 #5）。
- **动效克制**：150–200ms 缓动，只用于面板切换与气泡入场，不做花哨过场。
- 圆角小而统一（4px / 8px 两档），阴影只给浮层。
- 深色主题 v0.4（roadmap），实现时颜色全部走 token 变量，先把底子留好。

## 2. 主窗口：三栏结构

```
┌──────┬──────────────────────┬────────────────────────────────────────┐
│  我  │ [搜索]        [+]   │ 张三   ● 在线 · 某某科技/研发部/后端组  ⋯ │
│      ├──────────────────────┼────────────────────────────────────────┤
│ 聊天 │ ● 张三        10:24 │              ┌──────────────────────┐   │
│ 聊天 │   好的，收到    (2)  │              │ 需求文档发我一下？     │   │
│      │ 讨论组          9:15 │              └──────────────────────┘   │
│ 通讯 │   李四: [图片]       │   ┌────────────────────┐               │
│ 通讯 │ ○ 王五        昨天  │   │ [文件] 需求文档v3.docx │            │
│ 录   │   [文件] 设计稿.zip  │   │ 2.4 MB   已送达      │               │
│      │                      │   └────────────────────┘               │
│      │                      ├────────────────────────────────────────┤
│ 设置 │                      │  表情 截图 图片 文件                     │
│      │                      │  输入消息…                              │
└──────┴──────────────────────┴────────────────────────────────────────┘
  56px      250px（可拖拽）              自适应，≥560px
```

| 栏 | 内容 | 规格 |
|---|---|---|
| ① 导航栏 | 上：我的头像（点击=编辑资料）→ 聊天 → 通讯录；下：设置 | 56px 固定；茶青侧栏保持不变；聊天/通讯录主入口图标 25px，设置图标 21px；图标带未读总角标 |
| ② 列表栏 | 随导航切换：会话列表 / 通讯录；顶部搜索框常驻 | 250px，可拖 200–320px |
| ③ 内容区 | 聊天界面 / 联系人资料卡 / 空状态 | ≥560px；未选会话时显示淡灰 logo 空状态 |

- 窗口最小 960×640，记忆上次大小与位置；关闭按钮默认最小化到托盘（设置可改）。
- **沉浸式无标题栏**（决议 #49/#51/#52）：主窗与设置窗不显示系统标题栏，三栏内容直通窗口顶部。macOS 保留原生红绿灯，主窗红绿灯置于**列表栏顶部留白区（x=68）与搜索框左对齐**——56px 导航栏放不下三钮，不允许横跨分界线（决议 #51）；设置窗红绿灯在侧栏左上 (12,10)。Windows / Linux 在右上角自绘窗口控制按钮（最小化 / 最大化·还原 / 关闭，关闭悬停红底白字）。窗口顶部 32px 为隐形拖拽带（可拖动、双击最大化还原）：macOS / Windows 用系统 / CSS 拖拽区，**Linux 因 CSS 拖拽区不可靠改为 JS 拖拽**（决议 #52，对用户行为无差异）。导航栏头像、列表搜索框、聊天头部内容均从拖拽带下方开始；全屏弹层（向导、转发、图片查看器等）层级高于拖拽带，交互不受影响。

## 3. 列表栏 · 会话列表（导航=聊天）

- 条目：头像（带在线小圆点）、名称、最后消息摘要、时间、未读角标；免打扰会话角标改为静默小灰点。
- 摘要规则：`李四: [图片]`、`[文件] 设计稿.zip`、草稿优先显示红字 `[草稿] …`。
- 排序：置顶在前，其余按最后消息时间倒序（微信同款）。
- 右键菜单：置顶 / 免打扰 / 标为已读 / 从列表移除（不删历史，下次来消息会回来）。
- 顶部 `[+]`：发起讨论组（多选联系人，见 §7.1）。

## 4. 列表栏 · 通讯录（导航=通讯录）

```
[搜索]
▾ 讨论组
   # 产品讨论组 (8人)
▾ 某某科技 (23/41 在线)
   ▾ 研发部 (12/18)
      ▾ 后端组 (5/7)
         ● 张三  Windows  ← 在线
         ● 李四  macOS
         ○ 王五 · 离线     ← 灰显 + "离线"副文字，沉底
      ▸ 前端组 (4/6)
   ▸ 设计部 (3/8)
▸ 未分组 (2/5)
```

- **三级树：公司 ▸ 部门 ▸ 团队 ▸ 成员**，逐级折叠，每级标题显示（在线/总数）；资料字段空缺的层级自动跳过，全空归入"未分组"。
- 组内排序：在线优先，再按拼音；离线者灰显**且昵称旁加"离线"副文字**（不只靠颜色区分，决议 #17/补充轮）。
- **历史联系人持久保留**：凡是出现过的节点永久留在通讯录（离线灰显），重启不丢。不提供手动删除（对方还在网内时删了也会回来）；"清理 90 天未上线"放设置-高级。
- **资料自动刷新（防"机器换人"）**：对方上线时、改资料时即时更新；在线期间心跳每 30s 携带资料版本号，发现版本变化立刻拉取最新资料——机器没换、用的人换了昵称/部门，最迟半分钟内通讯录就纠正过来。
- 单击成员 → 内容区显示**联系人资料页**：顶部为圆形头像、昵称/备注、在线状态与组织路径；中部按“备注 / 详细信息”分组展示本地备注输入、IP、主机、平台、最近在线时间；底部提供「保存备注」「发消息」动作。资料页使用内容区完整页面布局，靠留白和细分割线组织信息，不使用突兀的居中小浮卡。双击成员 → 直接进入该联系人的单聊界面（复用打开会话探活，决议 #40）。
- 资料页可**设置备注名**（仅本机生效，列表与搜索优先按备注显示，决议 #22/#40）。
- 顶部固定「讨论组」分组，列出已加入的组。

## 5. 内容区 · 聊天界面

**头部**：单聊显示对方昵称，昵称下方以灰色小字显示对方 IP，右侧仍保留在线状态；点击昵称/IP 这块用户名字区域，在头部下方弹出紧凑资料框，展示头像、备注/昵称、组织、IP、主机、平台、在线状态，并可直接修改对方备注。备注仅本机可见，保存后会同步影响会话列表、通讯录和搜索显示。群聊显示组名 + 人数。**打开会话即触发一次探活**，约 2s 内以真实结果刷新状态，离线则显示灰字"离线"——防止对着"假在线"的人发消息（F-DISC-8）。右侧 `⋯` 打开会话设置（置顶/免打扰/查看成员/清空本会话记录）。群聊点人数 → 右侧滑出成员面板（含「+ 添加成员」）。

**消息区**：

- 气泡：对方靠左白底，自己靠右主题色浅底（比微信的纯色绿更轻，显高级）；圆角 8px。群聊中对方消息在左侧显示发送者圆形头像，私聊消息不在气泡旁显示头像。
- 时间分隔：间隔 >5 分钟插一条灰色小字时间；跨天插日期。
- 文本消息：链接自动识别可点；仅气泡本身可右键=复制 / **转发**（选目标会话，P1）/ 撤回（自己的、2 分钟内，P1），消息行空白区域不触发菜单；右键菜单靠近窗口边缘时自动避让，不被裁切。
- 图片消息：等比缩略图（最大 280×280）内嵌显示，点击进大图查看器（缩放/另存为）；右键菜单含**「转发」/「添加到表情」**。添加成功/失败以紧贴消息侧边、垂直居中的小胶囊反馈：对方消息显示在气泡右侧，自己消息镜像到气泡左侧。群聊图片显示在群会话内，发送端一条气泡汇总给在线成员的多条点对点传输；群聊图片超过 10MB 时退化为普通文件卡片，接收者手动接收。
- 表情包消息：固定 120px 高度渲染（不随原图大小变化），右键可「转发」或「添加到表情」收藏对方发的表情，添加反馈沿用图片消息的侧边胶囊。
- **文件消息卡片**（微信式）：

```
┌─────────────────────────┐      状态流转：
│ [文件图标] 设计稿.zip    │      等待对方接收 → 传输中(进度条+速率+取消)
│          48.2 MB         │      → 已完成(打开 / 打开所在文件夹)
│          6.2 MB/s        │      → 已取消 / 失败(重试)
└─────────────────────────┘
```

- 发送状态：发送中=自绘小转圈 → 已送达=自绘对勾；失败=红色 ! 点击重发；**对方离线=自绘时钟图标**，气泡下灰字"对方上线后自动送达"。
- 系统提示行（居中灰小字）：撤回提示、群成员变更等，尽量少，不刷屏。消息撤回后原气泡从本地消息流中隐藏，只保留"你撤回了一条消息 / 对方撤回了一条消息"提示。

**输入区**：工具条（表情 / 截图 / 图片 / 文件 / 文件夹 / 历史搜索，全部用自绘线性 SVG）+ 多行输入框 + 发送。历史搜索入口固定在工具条最右侧；工具条图标悬停约半秒后以淡入浮层显示中文名称，禁用态也保留名称提示。输入框占位 hint 使用比时间戳更浅一阶的 `--text-placeholder`，只起辅助提醒作用，不与已输入正文竞争。输入框编辑态与消息展示态使用同一套 Twemoji 本地 SVG 兼容渲染：当草稿包含内置 emoji 子集时，textarea 上方显示镜像渲染层，底层仍保留原生输入、选区、粘贴和发送行为。群聊中图片/文件/文件夹按钮在至少一位其他群成员在线时可用，发送后只投递给在线成员；离线成员不显示为待补发。

- 表情面板双页签：**emoji** / **我的表情包**。emoji 页显示 Twemoji 本地 SVG 图标，点击仍向输入框插入对应 UTF-8 emoji 字符；输入框编辑态与消息气泡正文展示时对内置 emoji 子集替换为同套 SVG，避免 Win7 / UOS 显示方框。表情包来源：聊天里任意图片右键「添加到表情」，**保存时自动压缩**（静图最长边 512px 转 WebP，GIF 动图 ≤2MB 原样保留）；面板内右键删除、拖拽排序；点击即发送。面板不会因鼠标移出自动关闭，仅在点击面板与表情按钮以外区域时关闭；切换 emoji / 表情包页签或表情包为空时，弹出框尺寸保持稳定。
- Enter 发送、Ctrl/Cmd+Enter 换行（设置可互换）。
- 群聊中输入 `@` 弹出成员选择浮层，被 @ 者收到加强通知（会话列表标"[有人@我]"红字，P1，决议 #22）。
- 粘贴图片 → 输入框出现预览块再发送；**拖文件/文件夹到聊天区任意位置即发送**（拖入时全区高亮蒙层提示）。

## 6. 全局搜索（微信式）

两个列表页顶部的搜索框是同一套组件，聚焦后弹出结果浮层，分类展示：

| 分类 | 范围 | 说明 |
|---|---|---|
| 联系人 | **全量**（含离线、含未分组） | 两个页签都能搜到所有人 |
| 讨论组 | 组名、成员名 | |
| 聊天记录 | 消息正文关键词 | 按会话聚合："与张三的聊天 · 3 条相关"，点击 → 进入会话并定位高亮该条 |
| 文件 | 收/发过的文件名 | 结果项可「打开」或「定位到聊天」 |

中文按字匹配、连续词组优先（基于本地索引，秒出结果；实现细节归技术设计）。

### 6.1 会话内历史搜索

群聊与私聊的输入区工具条最右侧显示“历史搜索”图标按钮。点击后弹出设置页尺度的居中面板，范围固定为当前会话；再次点击、切换会话、点击面板外遮罩或右上角关闭按钮均可关闭。面板宽高参考设置页的桌面软件感，优先保证筛选区、日期区、结果区稳定可扫描，不遮挡输入框内部控件。

面板内容：左侧为关键词输入框、“全部 / 图片 / 文件”分段筛选与连续日期范围选择，右侧为结果列表。打开面板时即显示当前会话最近记录；未输入关键词时按当前类型与日期筛选直接列出匹配记录。日期筛选使用单月日历组件：第一次点击选起始日，第二次点击结束日并高亮连续区间；再次点击可重新选择范围，清空筛选会一并清除日期。文本结果显示摘要，文件结果显示文件名，图片结果必须展示图片缩略图而不是只显示图片名；缩略图不可用时才回退为图片占位。结果项显示发送者与时间，点击后进入对应历史位置并高亮消息。

## 7. 关键流程

### 7.1 发起讨论组（参考微信：从单聊加人）

单聊头部 `⋯` →「添加成员」，或会话列表 `[+]` → 弹出**两步建群弹窗**。

第 1 步是联系人快速搜索与多选：顶部输入框用于按备注/昵称/公司/部门/团队/IP 即时过滤联系人，下面是结果列表（复选框 + 在线点 + 离线副文字），底部显示已选人数与已选成员条。几百人规模下不要求用户靠滚动定位联系人。

第 2 步是组设置：组名可留空，默认使用"张三、李四、王五的讨论组"；管理密码可留空，若填写则必须再输入一次确认一致，并可填写密码提示。若设置管理密码，成员管理群时需输入密码并能看到提示；若留空，仅创建 IP 可管理。

### 7.2 接收文件

对方发文件 → 我方聊天里出现文件卡片（「接收」「另存为」「拒绝」）→ 接收后显示进度 → 完成后卡片变为「打开 / 打开所在文件夹」。群聊文件卡片同样出现在群会话里，发送端一条卡片汇总各在线成员的接收进度。**单聊 ≤20MB 图片、群聊 ≤10MB 图片免确认**，直接以图片形式展示；群聊超过 10MB 的图片按普通文件卡片展示并手动接收。会话不在前台时走通知 + 角标。

### 7.3 托盘 / 菜单栏与通知（点击直达会话）

- 新消息：托盘 / 菜单栏 tooltip 始终显示未读总数；macOS 菜单栏图标旁显示未读数字，并同步 Dock 角标；Windows 使用任务栏 overlay 数字，同时托盘图标在原图标与带数字角标图标之间闪烁；Linux 尝试系统角标（Unity 支持时生效），默认以托盘图标闪烁作为跨桌面环境兜底。未读清零后所有数字与闪烁停止。
- **点击通知或托盘 → 主窗弹出并直接定位到对应会话**（用户原话需求）。
- 托盘右键：打开主窗 / 全局免打扰开关 / 退出。

### 7.4 首次启动向导（决议 #22）

首次打开必经三步，防止全网一片"未命名"：① 昵称（必填，默认取系统用户名）→ ② 公司 / 部门 / 团队（选填，可跳过）→ ③ 文件保存目录确认 → 进主界面。中途退出则下次启动重新进入向导。

### 7.5 给离线的人发消息

正常输入发送 → 气泡时钟态 + 会话内一次性灰字提示"对方当前离线，将在其上线后自动送达"→ 对方上线后自动补发，气泡转 ✓。

## 8. 设置页

独立设置窗口（640×480，微信式），左侧分组导航，右侧为「页头说明 + 分组面板 + 设置行」。设置窗同主窗为沉浸式无标题栏（决议 #49）：顶部 32px 隐形拖拽带，macOS 红绿灯内嵌，Windows / Linux 右上角仅自绘关闭按钮。设置页不按实现先后排列，而按用户心智与常见桌面软件规范组织：先身份与外观，再消息、聊天文件、网络，最后快捷键、高级和关于。每个设置行包含标题、说明、控件；禁用项必须说明原因或展示为后续能力。

**头像编辑器**（决议 #50）：左侧 64px 圆形大预览 + 当前样式小字（如「图标 · 茶青」/「昵称首字 · 湖蓝」）；右侧顶部「图标头像 / 昵称首字」分段切换；下方分两节——"图标"网格（10 列 × 2 行，30px 圆格，悬停轻微放大，选中主色描边）与"背景色"色板（24px 圆点，选中描边 + 中心白点）。点击任意图标或色板自动切回图标模式；昵称首字模式下背景色按昵称散列，与通讯录展示一致。

| 分组 | 设置项 |
|---|---|
| 账号资料 | 头像（编辑器版式见上：大预览 + 分段切换 + 20 个 Twemoji 图标网格 + 背景色板）、昵称、公司、部门、团队 |
| 通用外观 | 开机自启（默认开）；关闭主窗口行为：最小化到托盘（默认）/退出；主题；字体缩放 100%/110%/125%；语言（置灰，远期） |
| 消息通知 | 新消息系统通知开关；通知是否显示消息内容（隐私遮罩）；提示音（默认关，三款自定义试听留 v1.0 打磨）；发送键 Enter / Ctrl+Enter 互换 |
| 聊天与文件 | 文件保存位置（路径 + 浏览 + 保存）；图片自动接收策略；聊天记录导出（会话 + 时间范围 + HTML/TXT/迁移备份包）；导入备份包；传输记录空态/列表 |
| 网络连接 | 手动节点 IP；跨网段 CIDR 扫描；已保存节点/网段以可移除标签展示；扫描结果即时提示 |
| 快捷键 | 截图、显示/隐藏主窗：**录制式输入**（聚焦后直接按组合键填入，Esc/退格清空 = 禁用，决议 #57）；注册失败（被系统占用，如 UOS 深度截图占 Ctrl+Alt+A）在设置行下方红字提示；「恢复默认」一键回出厂组合；截图时隐藏茶话间窗口开关 |
| 安全与高级 | UDP/TCP 端口（改后提示全员需一致并重启生效）；监听网卡（默认全部）；诊断日志导出；90 天未上线联系人清理（后续） |
| 关于 | 版本号；运行时内核；纯内网/无遥测说明；开源许可；Twemoji 图形署名 |
