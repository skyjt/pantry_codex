## 1. 设计基调

**参考微信桌面版的设计语言**：三栏结构、大量留白、极轻的分割线、克制的色彩。"简约优雅、克制但不老气"落到具体手段上：

- **一个主题色**点缀全局（按钮、选中态、自己的气泡底色），其余全部中性灰阶；主题色定为**茶青 `#3D8B6B`**（决议 #1）。
- **系统字体栈**：`PingFang SC / Microsoft YaHei / Noto Sans CJK`，正文 14px，绝不用衬线、不用艺术字。
- **图标统一为项目内自绘线性 SVG**（1.6px 描边、圆角端点、继承当前文字色），不混用拟物/彩色图标，不用 emoji 充当系统图标；设置图标必须画成清晰齿轮，不使用放射状“太阳”隐喻。Win7 / UOS 缺字平台不可靠，头像和内置 emoji 子集的可见形态使用 Twemoji 本地 SVG 子集兼容渲染，输入框编辑态也必须覆盖，协议与复制仍保留原始 emoji 字符。
- **品牌 logo 使用统一的“茶杯气泡”标识**：杯身兼具对话气泡轮廓，右侧保留杯把，左下伸出对话尾。**v0.15.9 起以用户提供的 SVG 套件作为唯一品牌源（决议 #107）**：任务栏/Dock、标准 logo、小尺寸 logo、菜单 logo、macOS 菜单栏 outline、深浅背景和横向 logo 都保留为本地 SVG；应用内 `PantryBrandLogo` 直接加载 SVG，托盘和打包图标也从 SVG 光栅化生成，不再手写茶杯轮廓。历史背景：v0.5.28 曾改为饱满实心填充风（决议 #64），v0.15.6/#104 与 v0.15.8/#106 为参考图手绘过渡版。
- **动效克制**：150–200ms 缓动，只用于面板切换与气泡入场，不做花哨过场。
- 圆角小而统一（4px / 8px 两档），阴影只给浮层。
- 深色主题 v0.4（roadmap），实现时颜色全部走 token 变量，先把底子留好。

## 2. 主窗口：三栏结构

```
┌──────┬──────────────────────┬────────────────────────────────────────┐
│  我  │ [搜索]        [+]   │ (头像) 张三 · 在线 · 某某科技/研发部/后端组 ⋯ │
│      ├──────────────────────┼────────────────────────────────────────┤
│ 聊天 │ ● 张三        10:24 │              ┌──────────────────────┐   │
│ 聊天 │   好的，收到    (2)  │              │ 需求文档发我一下？     │   │
│      │ 讨论组          9:15 │              └──────────────────────┘   │
│ 通讯 │   李四: [图片]       │   ┌────────────────────┐               │
│ 通讯 │ ○ 王五        昨天  │   │ [文件] 需求文档v3.docx │            │
│ 录   │   [文件] 设计稿.zip  │   │ 2.4 MB   已送达      │               │
│      │                      │   └────────────────────┘               │
│      │                      ├────────────────────────────────────────┤
│ 设置 │                      │  表情 截图 图片 文件                     │
│      │                      │  输入消息…                              │
└──────┴──────────────────────┴────────────────────────────────────────┘
  56px      250px（可拖拽）              自适应，≥560px
```

| 栏 | 内容 | 规格 |
|---|---|---|
| ① 导航栏 | 上：我的头像（点击=编辑资料）→ 聊天 → 通讯录；下：刷新全局用户 → 设置 | 68px 固定（决议 #68，为容纳 mac 红绿灯由 56px 加宽）；**中性浅灰底（决议 #70，微信式，`--rail-bg`，右细线分隔列表栏）——图标默认深灰、选中/hover 茶青高亮**，让 mac 红绿灯落在浅色上自然；聊天/通讯录主入口图标 25px，设置图标 21px；图标带未读总角标 |
| ② 列表栏 | 随导航切换：会话列表 / 通讯录；顶部搜索框常驻 | 250px，可拖 200–320px |
| ③ 内容区 | 聊天界面 / 联系人资料卡 / 空状态 | ≥560px；未选会话时显示品牌 logo 空状态 |

- 窗口最小 960×640，记忆上次大小与位置；关闭按钮默认最小化到托盘（设置可改）。
- **沉浸式无标题栏**（决议 #49/#51/#52）：主窗与设置窗不显示系统标题栏，三栏内容直通窗口顶部。macOS 保留原生红绿灯，**主窗红绿灯在导航栏左上角 (9,7)**（贴合窗口圆角视觉中心）（决议 #68：导航栏由 56px 加宽至 68px 容纳标准红绿灯，回到微信式三栏左上角位置；此前决议 #51 曾临时移到列表栏顶部 x=68，与 UI 不协调已废弃）；设置窗红绿灯在侧栏左上 (12,10)。Windows / Linux 在右上角自绘窗口控制按钮（最小化 / 最大化·还原 / 关闭，关闭悬停红底白字）。窗口顶部 32px 为隐形拖拽带（可拖动、双击最大化还原）：macOS / Windows 用系统 / CSS 拖拽区，**Linux 因 CSS 拖拽区不可靠改为 JS 拖拽**（决议 #52，对用户行为无差异）。导航栏头像、列表搜索框、聊天头部内容均从拖拽带下方开始；全屏弹层（向导、转发、图片查看器等）层级高于拖拽带，交互不受影响。
- **刷新全局用户按钮（决议 #115）**：放在导航栏底部工具组、设置按钮正上方，使用 `refresh` 线性图标。仅当设置里至少有一个扫描网段时可点；点击后按钮进入加载态并禁用，图标外圈显示环形进度（决议 #162，由原下挂直条改为环绕图标的 conic 细环），进度代表“已发出探测地址 / 总地址”。完成后短暂停留再淡出，避免把左侧栏变成常驻状态面板。
- **左侧导航悬停说明（决议 #116/#117/#118/#119）**：聊天、通讯录、刷新全局用户、设置按钮均使用自绘中文 tooltip；tooltip 不得由 CSS `:hover` / `:focus-visible` 直接触发，必须在鼠标于按钮上真实移动后再计时约半秒显示，pointer 离开立即清理，避免应用启动、默认焦点或静止鼠标位置导致误弹。不使用原生 `title`，避免系统黄框和设计语言冲突。tooltip 采用单体圆角深色标签，不做箭头、不做位移，显隐只保留短透明度变化，避免显示/消失时出现“菱形 + 矩形”的拆件感。自己头像悬停时不显示普通 tooltip，而是在头像右侧显示个人信息卡；信息卡必须仅由鼠标悬停头像触发，启动焦点或键盘焦点不得误弹。卡片结构为头像 + 昵称 + 本机资料标签、组织路径行、主机/节点/端口分组资料，保留单行截断、足够留白、8px 圆角和克制阴影，移出头像即收起。

## 3. 列表栏 · 会话列表（导航=聊天）

- 条目：头像（带在线小圆点）、名称、最后消息摘要、时间、未读角标；免打扰会话仍显示未读数量，但角标改为灰色，区别于普通红色强提醒。
- **选中态**（决议 #71，微信式）：当前打开的会话整条**实心茶青底 + 白字**（名称白、时间/预览/旗标半透白、未读角标白底茶青字），一眼看清在哪个会话；hover 用淡灰区分，避免选中态与悬停混淆。
- 摘要规则：`李四: [图片]`、`[文件] 设计稿.zip`、`[PK] 猜拳` / `[PK] 骰子`（不展示结果）、草稿优先显示红字 `[草稿] …`。
- 排序：置顶在前，其余按最后消息时间倒序（微信同款）；**置顶会话整条加淡灰底色**（`--bg-pinned`，决议 #126），与普通会话一眼区分，hover / 选中态仍优先覆盖。
- 右键菜单：置顶 / 免打扰 / 移除会话（**决议 #125**：点击先弹二次确认弹窗，说明「移除后聊天记录将被删除」；确认后会话立即从列表移除并删除聊天内容，但底部居中浮出「已删除与『X』的聊天记录」+「撤回」按钮，**倒计时 10 秒**；10 秒内点「撤回」立刻恢复（期间未落库），超时才真正删除聊天记录（消息 + 全文索引 + 会话条目）。早期"只移出列表、不删历史"的行为已废弃）。
- 顶部 `[+]`：发起讨论组（多选联系人，见 §7.1）。

## 4. 列表栏 · 通讯录（导航=通讯录）

```
[搜索]
▾ 讨论组
   # 产品讨论组 (8人)
▾ 某某科技 (23/41 在线)
   ▾ 研发部 (12/18)
      ▾ 后端组 (5/7)
         ● 张三  Windows  ← 在线
         ● 李四  macOS
         ○ 王五 · 离线     ← 灰显 + "离线"副文字，沉底
      ▸ 前端组 (4/6)
   ▸ 设计部 (3/8)
▸ 未分组 (2/5)
```

- **三级树：公司 ▸ 部门 ▸ 团队 ▸ 成员**，逐级折叠，每级标题显示（在线/总数）；资料字段空缺的层级自动跳过，全空归入"未分组"。
- 组内排序：在线优先，再按拼音；离线者灰显**且昵称旁加"离线"副文字**（不只靠颜色区分，决议 #17/补充轮）。
- **历史联系人持久保留**：凡是出现过的节点永久留在通讯录（离线灰显），重启不丢。不提供手动删除（对方还在网内时删了也会回来）；"清理 90 天未上线联系人"为后续版本能力（决议 #150 起不再摆置灰占位，做好再加）。
- **资料自动刷新（防"机器换人"）**：对方上线时、改资料时即时更新；在线期间心跳每 30s 携带资料版本号，发现版本变化立刻拉取最新资料——机器没换、用的人换了昵称/部门，最迟半分钟内通讯录就纠正过来。
- 单击成员 → 内容区显示**联系人资料页**：顶部为圆形头像、昵称/备注、在线状态与组织路径；中部按“备注 / 详细信息”分组展示本地备注输入、IP、主机、平台、最近在线时间；底部提供「保存备注」「发消息」动作。资料页使用内容区完整页面布局，靠留白和细分割线组织信息，不使用突兀的居中小浮卡。双击成员 → 直接进入该联系人的单聊界面（复用打开会话探活，决议 #40）。
- 资料页可**设置备注名**（仅本机生效，列表与搜索优先按备注显示，决议 #22/#40）。
- 顶部固定「讨论组」分组，列出已加入的组。

## 5. 内容区 · 聊天界面

**头部**：单聊在头部显示对方头像（右下角在线/离线状态点）+ 昵称，昵称下方以小字显示在线/离线状态 · 对方 IP（在线茶绿、离线灰，决议 #81）。IP 是内网排障关键信息，副标题必须单行完整展示，不使用省略号截断（决议 #88）。点击头像/昵称这块用户名字区域，在头部下方弹出紧凑资料框，展示头像、备注/昵称、组织、IP、主机、平台、在线状态，并可直接修改对方备注。备注仅本机可见，保存后会同步影响会话列表、通讯录和搜索显示。群聊显示组名 + 人数。聊天头部与左侧列表搜索栏**等高（84px）**、顶部都让出 32px 拖拽带，底部分隔线跨栏连成一条，视觉统一（决议 #127）。**打开会话即触发一次探活**，约 2s 内以真实结果刷新状态，离线则显示灰字"离线"——防止对着"假在线"的人发消息（F-DISC-8）。右侧 `⋯` 打开会话设置（置顶/免打扰/查看成员/清空本会话记录）。群聊点人数 → 右侧弹出群信息面板（含「+ 添加成员」）。**面板覆盖右侧一整列**（绝对定位、从顶到底、盖在消息之上而非挤压消息，决议 #67）；成员每项显示紧凑圆形头像 + 在线/离线状态点（绿/灰，头像始终彩色，与私聊/会话列表一致；决议 #82 由原「离线灰显」改）。

**消息区**：

- 会话切换时消息区按会话恢复离开时的滚动位置；若离开时已在底部，返回后仍贴到最新。首次打开会话、通知/托盘/震动等后台入口直达会话、主窗隐藏后恢复并重新点开会话、以及点击“回到最新”时，默认滚到最新消息。当前会话新增消息时，仅当用户本来贴近底部或消息是自己发送时自动贴底，避免阅读历史时被打断（决议 #111）。
- 气泡：对方靠左白底，自己靠右主题色浅底（比微信的纯色绿更轻，显高级）；圆角 8px。群聊中对方消息在左侧显示发送者圆形头像，私聊消息不在气泡旁显示头像。
- 时间分隔：间隔 >5 分钟插一条灰色小字时间；跨天插日期。
- 文本消息：链接自动识别可点；仅气泡本身可右键=复制 / **转发**（选目标会话，P1）/ 撤回（自己的、5 分钟内；菜单项实时显示剩余「撤回（mm:ss）」、超时变灰为「撤回（超时）」，决议 #63），消息行空白区域不触发菜单；右键菜单靠近窗口边缘时自动避让，不被裁切。
- PK 消息：气泡本身右键仅保留「撤回」（自己的、5 分钟内，倒计时规则同文本）；不提供复制、转发或添加到表情。撤回后只隐藏这一条 PK 气泡，不影响别人之后另发的 PK 消息。
- 图片消息：等比缩略图（最大 280×280）内嵌显示，点击打开**独立图片窗口**（决议 #94/#95/#96/#97/#98/#99/#100/#101/#102/#103/#105），不在主聊天窗内强制覆盖，用户可继续聊天、搜索或拖动主窗口。图片窗口为普通可移动/可缩放桌面窗口，系统标题栏默认显示图片文件名并承担关闭能力；内容区为暗色画布，图片按初始规则居中展示：若原图宽高均小于当前屏幕工作区 70%，以 100% 原始尺寸打开；若宽或高超过 70% 阈值，则按先到阈值的边等比缩放到 70% 内并显示缩放比例。渲染时 `<img>` 的显示宽高必须显式等于 `naturalSize * zoom`，图片平面用窗口中心 `50% + offset` 定位，`transform` 只负责自身居中与旋转，避免大图/极端比例图片出现布局盒子与视觉尺寸不一致或溢出后贴左上。操作菜单为**底部半透明浮层**，覆盖在图片窗口下方，包含缩放比例、缩小 / 放大 / 适应窗口 / 原始大小 / 左旋 / 右旋 / OCR 识别或查看结果 / 复制全部 / 另存为，不再放内部关闭按钮；按钮 32px 热区、8px 圆角、茶青 active 态，OCR 识别中以同一菜单内的短状态文案展示进度。支持鼠标滚轮缩放、拖拽平移、双击在适应窗口与原始大小间切换（从适应放大到原始大小时以鼠标所在图片坐标为锚点），`Esc` 关闭，`+/-` 缩放，`0` 原始大小，`F` 适应窗口，`R` 旋转；加载失败时在中央显示轻量错误态。小图片加载后自动 OCR 但不自动弹窗，大图片需用户点击识别；用户点击底部 OCR 按钮时，若已有缓存或识别完成，则在图片窗口内打开**无全屏遮罩的文本结果窗**（右侧浮窗，标题“识别结果”，正文为原生 textarea，底部有复制全部与关闭）。用户在 textarea 内使用系统文本选择 / 复制，图片上不再显示 OCR token、命中框、文字光标、拖选高亮或局部复制按钮；OCR 不再接管图片上的 pointer 事件，图片拖拽、缩放、旋转始终按看图器本体处理。同一图片的 OCR 结果由主进程会话级内存缓存，图片重新打开后先恢复识别文本并可一键打开结果窗，不重新读取图片或跑 Tesseract。图片窗口使用 `pantry-img://transferId` 显示图片，另存为复用主进程保存对话框并挂在图片窗口上，不暴露本地路径；OCR 字节通过受限 `transferId` 读取，不展示本地路径。右键菜单含**「复制」/「转发」/「添加到表情」**；复制经主进程 Electron 原生剪贴板写入系统图片剪贴板，粘贴到输入框后按图片发送（决议 #137）。添加成功/失败以紧贴消息侧边、垂直居中的小胶囊反馈：对方消息显示在气泡右侧，自己消息镜像到气泡左侧。群聊图片显示在群会话内，发送端一条气泡汇总给在线成员的多条点对点传输；群聊图片超过 10MB 时退化为普通文件卡片，接收者手动接收。
- 表情包消息：固定 120px 高度渲染（不随原图大小变化），右键可「复制」/「转发」或「添加到表情」收藏对方发的表情；复制经主进程 Electron 原生剪贴板写入系统图片剪贴板，粘贴到输入框后按图片发送（GIF 表情复制为当前帧 PNG，完整动图复用转发或表情面板发送），添加反馈沿用图片消息的侧边胶囊。

```
│          48.2 MB         │      → 已完成(打开 / 打开所在文件夹)
│          6.2 MB/s        │      → 已取消 / 失败(重试)
└─────────────────────────┘
```

- **文件图标按扩展名彩色区分类型**（决议 #75）：Word / Excel / PPT / PDF / 压缩包 / 图片 / 音频 / 视频 / 文本 / 代码 / 安装包，其余回退通用文件图标；彩色徽标属“内容标识”，不受单色系统控件图标基线（§10）约束。
- **群聊文件接收进度**（决议 #75）：不再用整体进度条，改为「已接收 / 总数」x/x 计数（如 `10/20`）；鼠标移到 x/x 上浮出已接收成员名单，移出即收起。单聊文件仍用进度条 + 速率。
- **普通接收态文件卡（决议 #178）**：对方发来的普通文件在 `offering` 状态下，右侧不再纵向堆叠「接收 / 另存为 / 拒绝」。操作改为一行动作组：茶青主按钮「接收」，旁边两个 26px 图标按钮分别为文件夹图标「另存为」和 `x` 图标「拒绝接收」，两者必须有中文 tooltip / aria-label。卡片高度应接近普通完成态文件卡，不因接收动作变成高卡片。
- **私聊直接发送文件卡片（决议 #174/#176/#177）**：仅私聊支持，且必须先走普通文件发送，让发送方与接收方都看到普通文件卡。发送方自己的私聊普通文件卡在 `offering` 状态、文件信息已送达、对方在线且支持 `fd1` 时，在右侧操作区显示「直接发送」按钮；按钮使用茶青主按钮样式，离线 / 旧版本 / 尚未送达时置灰并给短提示。卡片保持紧凑三栏：等待状态不放进 meta 行，改为文件名同行固定状态片（「等待接收」/「发送中」），大小行只显示大小 / 文件数 / 速率，避免被右侧按钮挤成省略号；「直接发送」与取消动作同一行，取消在双动作场景下用小图标按钮。发送方完成态统一显示「发送成功」。接收方收到直接发送请求时，卡片不显示「接收 / 另存为 / 拒绝」，直接进入「接收中」状态；完成后只显示「已保存本地」，右侧动作保留「打开 / 打开所在文件夹」，不展示完整本地路径或发送人目录名。失败时发送方显示「直接发送失败」；接收方失败仍显示「继续」用于断点续传。群聊文件卡永远不展示「直接发送」按钮，也永远不进入直接发送状态。

- **PK 消息气泡（决议 #139/#140/#141/#142/#145）**：用于“猜拳 / 骰子”分歧解决，单聊与群聊一致。沿用既有气泡底色、8px 圆角、茶青主色与明暗主题 token，不做卡片嵌套；气泡为**横向卡**（决议 #145/#147，从原垂直三层 ~170px 瘦身到约 60px 高，避免顶掉过多聊天消息；卡片**固定宽度**——他人含参与按钮 200px、自己 132px，信息列 `flex:1` 撑满、结果文字左对齐、短文字右侧留白——避免不同结果「出了布 / 出了石头」或动画态「出拳中…」与定格态之间宽度忽长忽短）：**左侧**是固定 42px 的「统一开奖窗」——一个柔和下沉底色 + 内描边的圆角展示窗，骰子与猜拳的揭晓都在这一个等尺寸 / 圆角 / 底色的窗内进行，用统一画框消解差异：骰子是一枚 **CSS 3D 立方体**（28px），六面各画真实 1–6 点数，投掷时立方体翻滚、定格时转到结果点数那一面朝前；猜拳是一张 **3D 翻牌**（28px），正反双面同显本地 Twemoji 原色手势（石头 `👊`、剪刀 `✌️`、布 `✋`；石头用 `👊`(1f44a) 而非 `✊`(270a)——本地缺 `270a.svg` 资源，决议 #148），翻转轮播后停在结果手势正面。**右侧**为信息列：上行茶青小标题（同套线性图标 + 「猜拳」或「骰子」），下行是口语化结果文字——猜拳「出了石头 / 剪刀 / 布」，骰子「掷出 4 点」，不显示谁赢谁输。新发 / 新收消息播放一次约 1.5s 揭晓动画——立方体翻滚 / 卡片翻转快速进行后弹性减速落定到载荷结果，定格瞬间开奖窗闪一下茶青高亮；揭晓过程中结果文字固定为「投掷中…」（骰子）/「出拳中…」（猜拳），不随翻牌轮播跳动，定格后才显示「掷出 N 点」/「出了石头 / 剪刀 / 布」（决议 #146）。动画只用 transform / opacity，历史加载、翻页、重开会话与 `prefers-reduced-motion` 下直接显示最终结果（立方体 / 卡片直接定位到结果面，不转动）。收到他人的 PK 时，信息列右侧紧跟一颗参与按钮小胶囊（猜拳「我也来」、骰子「掷一下」，hover 实底茶青）；自己的 PK 不显示参与按钮。按钮有 hover / active / focus-visible / disabled 状态，文字不换行；点击发送一条新的独立 PK 消息，原按钮保留，允许反复点击。若当前没有可参与的在线对象（单聊对方离线，或群里只有自己在线），按钮保留但灰显，hover 提示「PK 只能和在线的人玩」。群聊不做排行榜或大结算面板。

- 发送状态：发送中=自绘小转圈 → 已送达=自绘对勾；失败=红色 ! 点击重发；**对方离线=自绘时钟图标**，气泡下灰字"对方上线后自动送达"。
- 系统提示行（居中灰小字）：撤回提示、窗口震动提示、群成员变更、群名变更等，尽量少，不刷屏。消息撤回后原气泡从本地消息流中隐藏，只保留"你撤回了一条消息 / 对方撤回了一条消息"提示。窗口震动提示为"你发送了一次窗口震动 / 对方发来一次窗口震动"（决议 #110）。群名变更提示格式为「你把群名「旧名」改成了「新名」」或「备注/昵称把群名「旧名」改成了「新名」」（决议 #87）。

**输入区**：工具条（表情 / PK / 窗口震动 / 截图 / 图片 / 文件 / 文件夹 / 历史搜索，全部用自绘线性 SVG 或明确文字符号）+ 多行输入框 + 发送。`PK` 入口放在表情之后、窗口震动之前，按钮热区与其他工具一致，由斜体 `PK` 字标组成（`skewX(-10)` 右倾 + 加粗描边、竞技动感，不带额外文字），热区与其他工具一致，tooltip 为「PK」。点击后在工具条上方弹出小浮层（左对齐按钮向右展开，避免浮层左侧被聊天面板 `overflow:hidden` 裁切），包含两个并排大按钮：与气泡开奖窗视觉呼应，同套线性图标在上、文字在下，文字分别为「猜拳」「骰子」。单聊对方离线、或群聊没有其他在线成员时，玩法按钮灰显，tooltip / 短反馈为「PK 只能和在线的人玩」；选择成功后立即关闭浮层并发送结构化 PK 消息，发送前不预览结果。私聊窗口震动入口仅在单聊且对方在线时可用，点击后发送一次即时震动；限流失败以工具条旁的轻量短文案反馈，发送成功则同时在消息区插入系统提示行。收端收到震动后唤起/震动主窗口，自动切到发起人的单聊，并在消息区插入系统提示行；若该单聊已设免打扰，只保留系统提示行，不唤起、不置前、不震动主窗口；不额外弹阻塞式对话框。历史搜索入口固定在工具条最右侧；工具条图标悬停约半秒后以淡入浮层显示中文名称，禁用态也保留名称提示。输入框占位 hint 使用比时间戳更浅一阶的 `--text-placeholder`，只起辅助提醒作用，不与已输入正文竞争。输入框编辑态与消息展示态使用同一套 Twemoji 本地 SVG 兼容渲染：当草稿包含内置 emoji 子集时，textarea 上方显示镜像渲染层，底层仍保留原生输入、选区、粘贴和发送行为。群聊中图片/文件/文件夹按钮在至少一位其他群成员在线时可用，发送后只投递给在线成员；离线成员不显示为待补发。剪贴板里的截图位图、以及从文件管理器复制的文件，均可直接 Ctrl+V / Command+V 粘贴到输入框发送（决议 #76）：单张图片文件按图片发送，其余按文件发送，与拖拽一致；若剪贴板同时包含文本和图片副本（如网页 / 富文本 emoji），文本优先走原生粘贴；无文本时先读浏览器 paste 图片项，仍为空则从主进程读取 Electron 原生图片剪贴板发送（决议 #135/#138）。主进程原生图片兜底必须晚于浏览器 paste 分流，并在同次 paste 已处理文件 / 文本 / 图片时取消，避免第三方截图工具同时暴露两路图片导致重复发送（决议 #180）。**输入框高度可手动拖拽调节（决议 #127）**：输入区顶部（消息/输入分隔线）有一条细灰手柄，hover 变深、`ns-resize` 光标，按住上下拖动改变输入框高度（约束 48–320px），向上拖更高、向下拖更矮；高度用本地存储记忆，纯渲染层、不影响工具条与发送行为。

- **文件发送入口（决议 #174）**：聊天工具条的「文件」和「文件夹」按钮保持现有普通发送路径，点击后直接打开系统选择器，不在发送前弹「直接发送」菜单。私聊发送成功后，若符合条件，再在发送方文件卡片上出现「直接发送」按钮；群聊文件 / 文件夹入口始终只走普通发送。

- 表情面板双页签：**emoji** / **我的表情包**。emoji 页显示 Twemoji 本地 SVG 图标，点击仍向输入框插入对应 UTF-8 emoji 字符；输入框编辑态与消息气泡正文展示时对内置 emoji 子集替换为同套 SVG，避免 Win7 / UOS 显示方框。表情包来源：聊天里任意图片右键「添加到表情」，**保存时自动压缩**（静图最长边 512px 转 WebP，GIF 动图 ≤2MB 原样保留）；面板内右键删除、拖拽排序；点击即发送。面板不会因鼠标移出自动关闭，仅在点击面板与表情按钮以外区域时关闭；切换 emoji / 表情包页签或表情包为空时，弹出框尺寸保持稳定。
- Enter 发送、Ctrl/Cmd+Enter 换行（设置可互换）。
- 群聊中输入 `@` 弹出成员选择浮层，被 @ 者收到加强通知（会话列表标"[有人@我]"红字，P1，决议 #22）。
- 粘贴图片 → 输入框出现预览块再发送；**拖文件/文件夹到聊天区任意位置即普通发送**（拖入时全区高亮蒙层提示）。私聊拖入文件不提供直接发送预选；发送成功后若符合条件，由发送方文件卡片显示「直接发送」按钮。群聊拖入文件只走普通发送，不出现直接发送入口。

两个列表页顶部的搜索框是同一套组件，聚焦后弹出结果浮层，分类展示：

| 分类 | 范围 | 说明 |
|---|---|---|
| 联系人 | **全量**（含离线、含未分组） | 两个页签都能搜到所有人 |
| 讨论组 | 组名、成员名 | |
| 聊天记录 | 消息正文关键词 | 按会话聚合："与张三的聊天 · 3 条相关"，点击 → 进入会话并定位高亮该条 |
| 文件 | 收/发过的文件名 | 结果项可「打开」或「定位到聊天」 |

中文按字匹配、连续词组优先（基于本地索引，秒出结果；实现细节归技术设计）。

### 6.1 会话内历史搜索

群聊与私聊的输入区工具条最右侧显示“历史搜索”图标按钮。点击后弹出设置页尺度的居中面板，范围固定为当前会话；再次点击、切换会话、点击面板外遮罩或右上角关闭按钮均可关闭。面板宽高参考设置页的桌面软件感，优先保证筛选区、日期区、结果区稳定可扫描，不遮挡输入框内部控件。

面板内容：左侧为关键词输入框、“全部 / 图片 / 文件”分段筛选与连续日期范围选择，右侧为结果列表。打开面板时即显示当前会话最近记录；未输入关键词时按当前类型与日期筛选直接列出匹配记录。日期筛选使用单月日历组件：第一次点击选起始日，第二次点击结束日并高亮连续区间；再次点击可重新选择范围，清空筛选会一并清除日期。

**结果项信息密度（决议 #74）**：以快速扫读为先。文本命中**直接以摘要为主体**（可换行至两行），不再显示“文本消息”小标题与前置类型图标；文件命中显示类型图标 + 文件名；图片命中必须展示图片缩略图而不是只显示图片名，缩略图不可用时才回退为图片占位。所有结果项底部显示发送者与时间。摘要中的内置 emoji 与消息展示一致，走 Twemoji 本地 SVG 彩色渲染，Win7 / UOS 不再出现方框或黑白线描。**命中交互**：文本与文件命中单击即跳转到对应历史位置并短暂高亮；**图片命中单击放大查看（复用大图查看器、不离开搜索面板），双击才跳转定位**，便于快速浏览图片而不丢失搜索上下文。

## 7. 关键流程

### 7.1 发起讨论组（参考微信：从单聊加人）

单聊头部 `⋯` →「添加成员」，或会话列表 `[+]` → 弹出**两步建群弹窗**。

第 1 步是联系人快速搜索与多选：顶部输入框用于按备注/昵称/公司/部门/团队/IP 即时过滤联系人，下面是结果列表（复选框 + 在线点 + 离线副文字），底部显示已选人数与已选成员条。几百人规模下不要求用户靠滚动定位联系人。

第 2 步是组设置：组名可留空，默认使用"张三、李四、王五的讨论组"；管理密码可留空，若填写则必须再输入一次确认一致，并可填写密码提示。若设置管理密码，成员管理群时需输入密码并能看到提示；若留空，仅创建 IP 可管理。

### 7.2 接收文件

对方发文件 → 我方聊天里出现文件卡片（右侧一行动作：「接收」主按钮 + 文件夹图标另存 + `x` 拒绝）→ 点击「接收」时默认保存到 `文件保存位置/联系人名称/`，点击文件夹图标「另存为」时保存到用户选择的目录 → 接收后显示进度 → 完成后卡片变为「打开 / 打开所在文件夹」。私聊直接发送自动接收也使用同一默认联系人子目录。群聊文件卡片同样出现在群会话里，发送端一条卡片汇总各在线成员的接收进度；接收方手动接收群文件时按发送人显示名进入联系人子目录。**单聊 ≤20MB 图片、群聊 ≤10MB 图片免确认**，直接以图片形式展示；群聊超过 10MB 的图片按普通文件卡片展示并手动接收。会话不在前台时走通知 + 角标。

### 7.3 托盘 / 菜单栏与通知（点击直达会话）

- 新消息：托盘 / 菜单栏 tooltip 始终显示未读总数；macOS 菜单栏图标旁显示未读数字，并同步 Dock 角标；Windows 使用任务栏 overlay 数字，同时托盘图标在原图标与带数字角标图标之间闪烁；Linux 尝试系统角标（Unity 支持时生效），默认以托盘图标闪烁作为跨桌面环境兜底。未读清零后所有数字与闪烁停止。
- 桌面通知摘要必须兼容 Win7 / UOS 缺字环境：通知正文不渲染 emoji 图形，文本里的内置 emoji / 常见 emoji 统一显示为 `[表情]`；表情包通知显示 `[表情]`，图片显示 `[图片]`，文件显示 `[文件] 文件名`，PK 显示 `PK · 猜拳` 或 `PK · 骰子` 且不展示结果。Linux / Windows 通知显式带真实应用图标，避免 UOS 通知中心显示系统默认图标（决议 #108/#139）。
- **点击通知或托盘 → 主窗弹出并直接定位到对应会话**（用户原话需求）。
- 托盘右键：打开主窗 / 全局免打扰开关 / 退出。

### 7.4 首次启动向导（决议 #22）

首次打开必经三步，防止全网一片"未命名"：① 昵称（必填，默认取系统用户名）→ ② 公司 / 部门 / 团队（选填，可跳过）→ ③ 文件保存目录确认 → 进主界面。中途退出则下次启动重新进入向导。向导首屏顶部使用真实品牌标识（`PantryBrandLogo` color 变体纯图形 + `茶话间 Teahouse` 中英文字标，与关于页 / 空状态一致，决议 #122/#155），不使用纯文字占位。第三步文案说明该目录用于存放收到的文件，不写"功能即将到来"——文件收发已是核心能力。

### 7.5 给离线的人发消息

正常输入发送 → 气泡时钟态 + 会话内一次性灰字提示"对方当前离线，将在其上线后自动送达"→ 对方上线后自动补发，气泡转 ✓。

### 7.6 局域网自更新（决议 #166）

主界面发现内网有**同平台、版本更高、可作更新源**的在线节点时，给一个**不打扰的提示入口**——主导航栏 / 列表栏顶部一枚淡茶青「内网有新版」徽标（带小红点），不弹窗打断当前操作；无更可用更新源时完全不显示。点开 → **更新弹窗**：显示新版本号、来源同事昵称、当前版本，操作「同步更新」；内网范围、不访问外网、索包不静默安装等机制说明收进圆形 `?` 帮助图标，hover / focus 时显示自绘 tooltip，不作为正文常驻。点「同步更新」→ 向该节点发起内网索包请求；当前实现仅提示请求中 / 已发出 / 暂不可用，不展示普通文件传输卡片。后续进度形态：后台从该节点拉取（复用文件传输的进度展示）→ 拉完校验通过提示「即将重启更新」→ 自动安装重启（Windows 静默完成、Linux 弹一次系统授权框）。失败给明确文案并可重试，不影响继续聊天。整套提示遵循 tokens 茶青单主色 / 8px 圆角 / 明暗主题 / `prefers-reduced-motion`。本期覆盖 Windows 安装版与 Linux deb，mac 暂缓。

## 8. 设置页

独立设置窗口（640×480，微信式），左侧分组导航，右侧为「页头说明 + 分组面板 + 设置行」。设置窗弹出时居中到主窗几何中心（决议 #123），而非屏幕中心——主窗被拖到副屏或角落时设置窗跟随主窗，并夹取到主窗所在显示器工作区防止越界。设置窗同主窗为沉浸式无标题栏（决议 #49）：顶部 32px 隐形拖拽带，macOS 红绿灯内嵌，Windows / Linux 右上角仅自绘关闭按钮。设置页不按实现先后排列，而按用户心智与常见桌面软件规范组织：先身份与外观，再通知、聊天文件、网络，最后快捷键和关于（**决议 #150 重组为 7 组**：端口归「网络」、发送键归「聊天与文件」、去掉「安全与高级」杂物抽屉）。每个设置行包含标题、说明、控件。**保存策略统一（决议 #150 / #151）**：所有设置**即时保存**——偏好（开关 / 主题 / 字体 / 提示音 / 发送键）调整即生效，文本输入（资料 / 端口 / 快捷键 / 文件位置）失焦(blur)即存、头像与目录选定即存；**不再有任何保存按钮**（仅保留「更改」浏览、「恢复默认」「重置应用设置」等动作按钮）。每次保存后底部居中弹出「设置已保存」深色胶囊 toast，淡入 + 轻微上移、停留约 1.8s 后淡出（决议 #151，尊重 `prefers-reduced-motion`）；值未变则跳过不弹，昵称空 / 端口越界则复原原值并提示原因。二选一偏好（主题、发送键）用分段控件而非下拉。视觉走**设置页专属标尺**：间距 4px 台阶、圆角 Shape Lock（容器 8 / 控件 6 / 胶囊 999）、字号 4 级阶梯（18/15/13/12，区块标题 15 与正文 13 拉开层级）。未实现能力不摆置灰占位，做好再加。

**头像编辑器**（决议 #50）：左侧 64px 圆形大预览 + 当前样式小字（如「图标 · 茶青」/「昵称首字 · 湖蓝」）；右侧顶部「图标头像 / 昵称首字」分段切换；下方分两节——"图标"网格（10 列 × 2 行，30px 圆格，悬停轻微放大，选中主色描边）与"背景色"色板（24px 圆点，选中描边 + 中心白点）。点击任意图标或色板自动切回图标模式；昵称首字模式下背景色按昵称散列，与通讯录展示一致。
